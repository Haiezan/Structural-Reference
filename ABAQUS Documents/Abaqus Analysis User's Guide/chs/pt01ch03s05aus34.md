# 3.5.3 Abaqus/Explicit 中的并行执行







**产品：**Abaqus/Explicit Abaqus/CAE  

##### **参考**

-["Obtaining information," Section 3.2.1](pt01ch03s02abx01.md)-["Using the Abaqus environment settings," Section 3.3.1](pt01ch03s03aus30.md)-["Controlling job parallel execution," Section 19.8.8 of the Abaqus/CAE User's Guide](../usi/usi-link.md#usi-ana-jobeditor-parallelizationbtn)

### 概述

Abaqus/Explicit 中的并行执行： 
- 减少需要大量增量的分析的运行时间；
- 减少包含大量节点和元素的分析的运行时间；
- 产生与用于分析的处理器数量无关的分析结果；
- 可用于使用基于线程的循环级别或基于线程的域分解实现的共享内存计算机；和
- 可用于使用基于 MPI 的域分解并行实现的共享内存计算机和计算机集群。

### 调用并行处理

Abaqus/Explicit 中的并行化通过两种方式实现：域级和循环级。域级方法将模型分解为拓扑域，并将每个域分配给处理器。域级方法是默认方法。循环级方法并行化负责大部分计算成本的低级循环。单元、节点和接触对操作占低级并行例程的大部分。

可以通过指定要使用的处理器数量来调用并行化。

| **输入文件用法：** |在命令行中输入以下内容： |
| --- | --- |
|  | **abaqus** **job**=*job-name* **cpus**=*n* 例如，以下输入将在两个处理器上运行作业“beam”：``` abaqus job=beam cpus=2 ```|

| **Abaqus/CAE 用法：** |作业模块：作业编辑器：**并行化**：打开**使用多个处理器**，并指定处理器数量，*n* |
| --- | --- |

### 域级并行化

域级方法将模型划分为多个拓扑域。这些域被称为并行域，以将它们与与分析相关的其他域区分开。这些域均匀分布在可用处理器之间。然后在每个域中独立地进行分析。然而，信息必须在每个增量中在域之间传递，因为域共享公共边界。域级方法支持 MPI 和基于线程的并行化模式。

在初始化期间，域级方法对模型进行划分，以便生成的域花费大约相同的计算量。负载平衡定义为最昂贵进程中所有域的计算费用与最便宜进程中所有域的计算费用之比。对于表现出严重负载不平衡的情况，无论是因为初始负载平衡不够（静态不平衡）还是因为不平衡随着时间的推移而发展（动态不平衡），可以应用动态负载平衡技术（请参阅["Abaqus/Standard, Abaqus/Explicit, and Abaqus/CFD execution," Section 3.2.2](pt01ch03s02abx02.md)）。动态负载平衡基于过度分解：用户选择多个域，其数量是处理器数量的倍数。在计算过程中，Abaqus/Explicit 将定期测量计算费用并在处理器上重新分配域，以最大限度地减少负载不平衡。动态负载平衡不支持以下功能： 
- 选择性再循环（["Selective subcycling," Section 11.7.1](pt04ch11s07aus75.md)）
- 联合仿真（["Co-simulation," Section 17.1](pt04ch17s01.md)）
- 使用结果文件 (["Predefined fields," Section 34.6.1](pt07ch34s06aus128.md)) 的预定义字段

动态负载均衡方案的效率取决于问题固有的负载不平衡、过度分解的程度以及硬件的效率。当域的数量是处理器数量的两到四倍时，大多数不平衡问题都会得到最佳的性能改进。在互连速度较慢的系统（例如千兆位以太网集群）上，效率可能会显着降低。当不需要外部互连时，例如在集群的多核节点内或在共享内存系统上，可以获得最佳结果。最有可能从动态负载平衡中受益的应用程序是具有强烈时间依赖性和/或空间变化计算负载的问题。例如，包含安全气囊的模型，其中接触碰撞活动高度局部化且依赖于时间；以及耦合的拉格朗日-欧拉问题，即当材料在空旷的空间中移动时，本构活动会跟随材料移动。

为每个域创建单元和节点集，并可以在 Abaqus/CAE 中进行检查。这些集合被命名为`domain_*n*`，其中 *n* 是域编号。

在分析过程中，将创建单独的状态 (`*job-name*.abq`) 和选定结果 (`*job-name*.sel`) 文件。每个处理器都会有一种状态和一个选定的结果文件。命名约定是将处理器编号附加到文件名中。例如，状态文件名为`*job-name*.abq.*n*`，其中 *n* 是处理器编号。分析完成后，各个文件会自动合并为单个文件（例如，`*job-name*.abq`），并删除各个文件。

| **输入文件用法：** |在命令行中输入以下内容： |
| --- | --- |
|  | **abaqus** **job**=*job-name* **cpus**=*n* **parallel**=`domain`**domains**=*m* **dynamic_load_balancing** 例如，以下输入将使用域级并行化方法在两个处理器上运行作业“beam”：``` abaqus job=beam cpus=2 parallel=domain domains=2 ```还可以使用环境文件参数在环境文件中设置域级并行化方法**并行**=`DOMAIN`和**域**。 |

| **Abaqus/CAE 用法：** |作业模块：作业编辑器：**并行化**：打开**使用多个处理器**并指定处理器数量*n*； **域数量：** *m*；开启**激活动态负载平衡**； **并行化方法**：**域** |
| --- | --- |
|  |当域数量是处理器数量的倍数时，您可以激活动态负载平衡。 |

#### 结果的一致性

分析结果与用于分析的处理器数量无关。然而，结果确实取决于域分解期间使用的并行域的数量。除了单域模型和多域模型由于多个并行域尚不可用的功能（如下所述）而不同的情况外，这些差异应该仅由有限精度效应触发。例如，节点力集合的顺序可能取决于并行域的数量，这可能导致计算的力的尾随数字存在差异。一些物理系统对小扰动高度敏感，因此在一个增量中施加的力的微小差异可能会导致后续增量中结果的显着差异。涉及屈曲和其他分叉的模拟往往对小扰动敏感。

为了在每次运行中获得一致的分析结果，域分解中使用的域数量应该是恒定的。增加域的数量会稍微增加计算成本；因此，除非应用动态负载平衡，否则建议将域数量设置为等于用于分析执行的最大处理器数量，以获得最佳性能。如果不指定域的数量，则该数量默认为处理器的数量。

#### 不允许域级并行化的功能

不允许使用具有以下特征的域级并行化方法：
- Extreme value output.
- 稳态检测。

如果包含这些功能，则会发出错误消息。

#### 不能跨域分割的功能

某些功能无法跨域拆分。域分解算法自动考虑到这一点，并强制这些功能完全包含在一个域内。如果创建的域少于请求的处理器，Abaqus/Explicit 会发出错误消息。即使算法成功创建了所请求数量的域，负载也可能会不均匀地平衡。如果此行为不可接受，则应使用循环级并行化方法运行作业。

自适应平滑域不能跨越平行域边界。自适应平滑域与非自适应域边界上的节点以及自适应平滑域表面上的自适应节点不能与其他并行域共享。为了在指定并行域时以一致的方式强制执行此操作，相邻自适应平滑域共享的所有节点将被设置为非自适应。在这种情况下，分析结果可能与没有并行域的串行运行的结果显着不同。将并行域的数量设置为 1，如果不需要这种行为，则切换到循环级并行化方法。详细信息请参见["Defining ALE adaptive mesh domains in Abaqus/Explicit," Section 12.2.2](pt04ch12s02aus78.md)。

接触对不能跨并行域分割，但单独的接触对不限于位于同一并行域中。使用运动学接触算法的接触对要求与所涉及的表面关联的所有节点都位于单个并行域内，并且不与任何其他并行域共享。使用惩罚接触算法的接触对要求关联节点是单个并行域的一部分，但这些节点也可以是其他并行域的一部分。如果使用接触对，则大量节点参与接触的分析可能无法很好地扩展，尤其是在接触约束的运动学强制执行的情况下。一般联系并不限制域分解的边界。

运动学约束 (["Kinematic constraints: overview," Section 35.1.1](pt08ch35s01abo32.md)) 中涉及的节点（基于表面的壳到实体约束除外）将位于单个并行域内；并且它们不会与另一个并行域共享。然而，不共享节点的两个运动学约束可以放置在不同的并行域中。

在某些情况下，共享节点的梁单元可能会被迫进入同一并行域。这种情况仅发生在质心与梁节点位置不重合的梁或具有附加惯性的梁上（请参见["Adding inertia to the beam section behavior for Timoshenko beams" in "Beam section behavior," Section 29.3.5](pt06ch29s03alm10.md#usb-elm-ebeamsectionbehavior-addinertia)）。

#### 重新启动

使用域级并行化时，重新启动有一定的限制。为了确保实现最佳并行加速，必须选择用于重新启动分析的处理器数量，以便原始分析期间使用的并行域数量可以在处理器之间均匀分布。因为域分解仅基于原始分析中指定的特征和其中定义的步骤，所以影响域分解的特征只有在使原始域分解无效时才被限制在重新启动步骤中定义。由于新添加的功能将添加到现有域中，因此可能会出现负载不平衡以及相应的并行性能下降。

重新启动分析需要将原始分析期间创建的单独状态和选定结果文件转换为单个文件，如["Abaqus/Standard, Abaqus/Explicit, and Abaqus/CFD execution," Section 3.2.2](pt01ch03s02abx02.md)中所述。这应该在原始分析结束时自动完成。如果原始分析未能成功完成，您必须在重新启动之前转换状态和选定的结果文件。打包为使用域级并行化技术运行的 Abaqus/Explicit 分析无法使用循环级并行化技术重新启动或继续。

#### 联合仿真

用于 Abaqus/Explicit 与 Abaqus/Standard 或第三方分析程序运行时耦合的联合仿真技术 (["Co-simulation: overview," Section 17.1.1](pt04ch17s01abo17.md)) 可以与串行或并行运行的 Abaqus/Explicit 一起使用。

### 循环级并行化

循环级方法并行化代码中的低级循环，这些循环负责大部分计算成本。使用循环级并行化的加速因子可能明显小于使用域级并行化所能实现的加速因子。加速因子将根据分析中包含的功能而变化，因为并非所有功能都使用并行循环。例如一般接触算法和运动学约束。根据分析，循环级方法对于四个以上处理器的扩展性可能很差。通过此方法使用多个并行域会降低并行性能，因此不建议使用。 Windows 平台不支持循环级方法。

此方法的分析结果不取决于所使用的处理器的数量。

| **输入文件用法：** |在命令行中输入以下内容： |
| --- | --- |
|  | **abaqus** **job**=*job-name* **cpus**=*n* **parallel**=`loop`也可以使用环境文件参数 **parallel**=`LOOP`在环境文件中设置循环级并行化方法。 |

| **Abaqus/CAE 用法：** |作业模块：作业编辑器：**并行化**：打开**使用多个处理器**，并指定处理器数量*n*； **并行化方法：循环** |
| --- | --- |

#### 重新启动

使用循环级并行化时，对于可包含在重新启动分析中定义的步骤中的功能没有任何限制。出于性能原因，重新启动时使用的处理器数量必须是原始分析中使用的处理器数量的一个因素。最常见的情况是使用与原始分析中使用的相同数量的处理器重新启动。打包为使用循环级并行化技术运行的 Abaqus/Explicit 分析无法使用域级并行化技术重新启动或继续。

### 测量并行性能

并行性能是通过将在单个处理器上运行（串行运行）所需的总时间与在多个处理器上运行（并行运行）所需的总时间进行比较来测量的。该比率称为加速因子。在完美并行化的情况下，加速因子将等于用于并行运行的处理器数量。缩放是指随着处理器数量的增加加速因子的行为。完美缩放表示加速因子随着处理器数量线性增加。对于这两种并行化方法，加速因子和扩展行为在很大程度上取决于问题。一般来说，域级方法将扩展到更多数量的处理器并提供更高的加速系数。

### 输出

没有输出限制。
