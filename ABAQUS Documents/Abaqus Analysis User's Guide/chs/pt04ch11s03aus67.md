# 11.3.1 在模型中引入几何缺陷

**产品：**Abaqus/Standard  Abaqus/Explicit

##### **参考**

- ["不稳定屈曲和后屈曲分析," 第 6.2.4 节](pt03ch06s02at03.md)
- [*IMPERFECTION](../key/key-link.md#usb-kws-mimperfection)

### 概述

几何缺陷模式：

- 通常为后屈曲载荷-位移分析引入模型；
- 可以定义为先前使用 Abaqus/Standard 完成的特征值屈曲预测或特征频率提取分析所得屈曲特征模态的线性叠加；
- 可以基于先前使用 Abaqus/Standard 完成的静态分析解；或
- 可以直接指定。

### 一般后屈曲分析

在 Abaqus/Standard 中，Riks 方法（["不稳定屈曲和后屈曲分析," 第 6.2.4 节](pt03ch06s02at03.md)）可用于求解具有稳定或不稳定后屈曲行为的后屈曲问题。但是，由于屈曲点处响应不连续（分叉），精确的后屈曲问题通常无法直接分析。要分析后屈曲问题，必须通过在“完美”几何中引入几何缺陷模式，使临界载荷达到之前就在屈曲模态中产生一定响应，从而将其转化为具有连续响应而不是分叉响应的问题。

### 引入几何缺陷

缺陷通常通过几何扰动引入。Abaqus 提供三种定义缺陷的方法：屈曲特征模态的线性叠加、来自静态分析的位移，或直接指定节点号和缺陷值。只有平移自由度会被修改。随后 Abaqus 将使用基于扰动坐标的常规算法计算法线。除非已知缺陷的精确形状，否则可以引入由多个叠加屈曲模态组成的缺陷（["特征值屈曲预测," 第 6.2.3 节](pt03ch06s02at02.md)）。

常用方法包括两个分析作业，二者使用相同的模型定义：用 Abaqus/Standard 确定可能的失稳模态，再用 Abaqus/Standard 或 Abaqus/Explicit 执行后屈曲分析。

1. 在第一次分析中，对“完美”结构使用 Abaqus/Standard 执行特征值屈曲分析，以确定可能的失稳模态并验证网格能够准确离散这些模态。将默认全局坐标系中的特征模态作为节点数据写入结果文件（["输出到数据和结果文件," 第 4.1.2 节](pt02ch04s01aus39.md)）。
2. 在第二次分析中，使用 Abaqus/Standard 或 Abaqus/Explicit 将这些屈曲模态添加到“完美”几何中，从而在几何中引入缺陷。最低阶屈曲模态通常被认为提供最关键的缺陷，因此通常对其缩放并添加到完美几何中，形成扰动网格。缺陷形式为 ![](../graphics/usb_eqn01800.gif)，其中 ![](../graphics/usb_eqn01801.gif) 为第 ![](../graphics/usb_eqn00147.gif) 阶模态形状，![](../graphics/usb_eqn01802.gif) 为相应比例因子。必须选择各模态的比例因子；通常，如果结构对缺陷不敏感，最低屈曲模态应具有最大因子。所用扰动幅值通常为相对结构尺寸（如梁截面或壳厚度）的几个百分点。
3. 使用 Abaqus/Standard 或 Abaqus/Explicit 执行后屈曲分析。在 Abaqus/Standard 中，使用 Riks 方法对包含缺陷的结构执行几何非线性载荷-位移分析。这样，如果“完美”结构在屈曲前呈线性行为，Riks 方法可用于对这类“刚性”结构执行后屈曲分析。通过执行载荷-位移分析，还可以包含材料非弹性或接触等其他重要非线性效应。在 Abaqus/Explicit 中，则对扰动后的结构执行后屈曲分析。

Abaqus 通过用户节点标签导入缺陷数据。Abaqus 不检查两次分析之间的模型兼容性。原始模型和带缺陷模型中的节点集定义可能不同。对于 Abaqus 会生成附加节点的模型（例如为 20 节点砖单元上的接触面生成的节点），必须确保两次分析的模型完全相同，并且生成节点的节点信息写入结果文件。

如果模型按部件实例装配定义，则需要原始分析的部件（`.prt`）文件来从结果文件读取特征模态。原始模型和后续模型都必须以部件实例装配的方式一致定义。

#### 基于特征模态数据定义缺陷

要基于加权模态形状的叠加定义缺陷，需要指定先前特征频率提取或特征值屈曲预测分析的结果文件和步骤。可选地，可以为指定节点集导入特征模态数据。

| **输入文件用法：** | ``` [*IMPERFECTION](../key/key-link.md#usb-kws-mimperfection), FILE=*results_file*, STEP=*step*, NSET=*name* ``` |
| --- | --- |

#### 基于静态分析数据定义缺陷

要基于先前静态分析（["不稳定屈曲和后屈曲分析," 第 6.2.4 节](pt03ch06s02at03.md)）的变形几何定义缺陷，需要指定先前静态分析的结果文件和步骤，并可选择指定增量号。如果未指定增量号，Abaqus 将从结果文件中指定步骤可用的最后一个增量读取数据。可选地，可以为指定节点集导入模态数据。

| **输入文件用法：** | ``` [*IMPERFECTION](../key/key-link.md#usb-kws-mimperfection), FILE=*results_file*, STEP=*step*, INC=*inc*, NSET=*name* ``` |
| --- | --- |

#### 直接定义缺陷

可以将缺陷直接指定为全局坐标系中节点号和坐标扰动的表格，也可以选择在圆柱或球坐标系中指定。也可以从单独的输入文件读取缺陷数据。

| **输入文件用法：** | ``` [*IMPERFECTION](../key/key-link.md#usb-kws-mimperfection), SYSTEM=*name*, INPUT=*input file* ``` |
| --- | --- |
|  | 如果未指定输入文件，Abaqus 假定数据跟在该选项之后。 |

### 缺陷敏感性

某些结构的响应强烈依赖于原始几何中的缺陷，特别是在屈曲后多个屈曲模态相互作用时。因此，基于单个屈曲模态的缺陷往往会得到非保守结果。通过调整各屈曲模态比例因子的大小，可以评估结构对缺陷的敏感性。通常应进行多次分析，以研究结构对缺陷的敏感性。具有许多相近特征模态的结构往往对缺陷敏感，与最低特征值模态对应的缺陷形状未必给出最不利情况。

如果缺陷较大，缺陷结构将更容易分析。如果缺陷较小，在临界载荷以下变形相对于缺陷会很小；响应将在临界载荷附近快速增长，并引入行为的快速变化。

另一方面，如果缺陷较大，后屈曲响应将在达到临界载荷前平稳增长。在这种情况下，进入后屈曲行为的过渡会更平滑，也相对更容易分析。

### 输入文件模板

以下示例说明了一个结构的后屈曲分析，该结构的缺陷由屈曲特征模态的线性叠加定义，并包含两个使用相同模型定义的分析作业。

第一次分析使用 Abaqus/Standard 执行特征值屈曲分析，以确定可能的失稳模态并将其写入结果文件。

```md
[*HEADING](../key/key-link.md#usb-kws-mheading)
*Initial analysis run to write the buckling modes to the results file*
[*NODE](../key/key-link.md#usb-kws-mnode)
*Data lines to define initial "perfect" geometry*
...
**
[*STEP](../key/key-link.md#usb-kws-hstep)
[*BUCKLE](../key/key-link.md#usb-kws-hbuckle)
*Data lines to define the number of buckling eigenmodes*
[*CLOAD](../key/key-link.md#usb-kws-hcload) and/or [*DLOAD](../key/key-link.md#usb-kws-hdload) and/or [*DSLOAD](../key/key-link.md#usb-kws-hdsload) and/or [*TEMPERATURE](../key/key-link.md#usb-kws-htemperature)
*Data lines to specify the reference load, ![](../graphics/usb_eqn00454.gif)*
[*NODE FILE](../key/key-link.md#usb-kws-hnodefile), GLOBAL=YES, LAST MODE=*n*
U
[*END STEP](../key/key-link.md#usb-kws-hendstep)
```

第二次分析引入缺陷，并在 Abaqus/Standard 中采用修正 Riks 方法执行后屈曲分析。

```md
[*HEADING](../key/key-link.md#usb-kws-mheading)
*Second analysis run to define the imperfection and perform the postbuckling analysis*
[*NODE](../key/key-link.md#usb-kws-mnode)
*Data lines to define initial "perfect" geometry*
...
[*IMPERFECTION](../key/key-link.md#usb-kws-mimperfection), FILE=*results_file*, STEP=*step*
*Data lines specifying the mode number and its associated scale factor*
...
**
[*STEP](../key/key-link.md#usb-kws-hstep), NLGEOM
[*STATIC](../key/key-link.md#usb-kws-hstatic), RIKS
*Data line to define incrementation and stopping criteria*
[*CLOAD](../key/key-link.md#usb-kws-hcload) and/or [*DLOAD](../key/key-link.md#usb-kws-hdload) and/or [*DSLOAD](../key/key-link.md#usb-kws-hdsload) and/or [*TEMPERATURE](../key/key-link.md#usb-kws-htemperature)
*Data lines to specify reference loading*, ![](../graphics/usb_eqn00454.gif)
[*END STEP](../key/key-link.md#usb-kws-hendstep)
```
